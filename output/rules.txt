CONSEQUENTIAL COMPONENTS: Button, AmountInput, and ConfirmSheet are consequential. Never recompose them from primitives.

AmountInput MUST be used for all currency amount entry in payment and transfer flows. Never substitute a generic TextInput.
Button MUST be used for all submit, confirm, and pay actions. Never build a button from primitives.

DO NOT pass empty strings for optional properties; omit the property entirely if the value is unknown.
ENUM VALUES ONLY: Every property with an enum constraint must receive exactly one of its listed values; no freeform strings.
StatCard value MUST be a pre-formatted string with units applied (e.g. "$12,400"); never pass raw numbers.
TransactionRow amount MUST be a formatted currency string including symbol; reflect debit/credit in direction, not in the amount text.
Badge text MUST reflect actual data state; never use placeholder text.
Never nest raised Cards inside raised Cards.
