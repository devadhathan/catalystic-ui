Documentation

Catalystic UI is a rendering layer for AI agents: your agent emits a component schema, and we render it as real, on-brand, interactive UI.

Overview

Instead of returning markdown or raw HTML, an agent returns a small JSON component tree (we call it A2UI). Catalystic UI validates that tree against a design-system catalog and renders it with real components — buttons, forms, charts, cards — in the design system you choose (shadcn, Polaris, Material, Carbon, or your own imported from Figma).

Because the model returns data and never executable code, an injected script can't run: only components that exist in the catalog render, and all text is inserted as text.

How it works

  • Intent → the user (or an upstream system) sends a message.
  • Schema → the agent returns an A2UI component tree.
  • Verify → the tree is checked against catalog.json; non-consequential mismatches are repaired, consequential ones are rejected.
  • Render → the tree is painted with the active design system, fully interactive.

Quickstart

Open the studio, describe what you want, and watch it render. To drive it programmatically, create an API key under Settings → API keys and call the generate endpoint:

# POST a prompt, get back a rendered component tree
curl https://playground-eta-eight.vercel.app/api/chat \
  -H "Authorization: Bearer ck_your_key" \
  -H "Content-Type: application/json" \
  -d '{"prompt": "a booking screen for a hair salon"}'

The A2UI schema

A surface is an array of component nodes. Each node has a type from the catalog and typed props; children may be nested inline.

{
  "type": "Card",
  "props": { "title": "Book an appointment" },
  "children": [
    { "type": "TextInput", "props": { "label": "Name" } },
    { "type": "Button", "props": { "label": "Confirm", "variant": "primary" } }
  ]
}

Catalog & verification

Every design system ships a catalog.json describing which components exist and the shape of their props. At render time the tree is verified against it:

  • Non-consequential issues (an unknown prop, a bad enum) are repaired so the screen still renders.
  • Consequential components — anything marked x-consequential, like Button, AmountInput, ConfirmSheet — are rejected if they don't validate, so a malformed action never renders.
The same catalog is enforced in the browser (AJV) and on the server (jsonschema), so client and server agree on what's valid.

Generate API

Authenticate with an API key (ck_…) in the Authorization header. Each real generation spends one credit; identical requests are served from cache for free.

FieldTypeNotes
promptstringA single user message. Use history for multi-turn.
historyarrayChat messages {role, content}.
componentsarrayThe current surface, for follow-up edits.
modestringchat (default) or an agent mode.
webSearchboolLet the agent pull live info.

Responses include reply, components, usage, and credits_remaining.

Bring your own agent

Point Catalystic UI at your own agent endpoint and it drives the renderer. Your endpoint receives the chat turn and must return { reply, components }. Requests are proxied over HTTPS only, with private/loopback hosts blocked.

POST /api/proxy
{ "endpoint": "https://your-api/agent",
  "body": { "history": [...], "mode": "chat" } }

Credits & billing

New accounts start with free credits. One credit is one fast generation; a higher-quality generation costs five. Cached (identical) renders are free. Top up any time from Settings → Usage & credits — checkout is handled by Stripe and credits are applied to your account the moment payment confirms.
